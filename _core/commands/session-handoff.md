# 커스텀 명령어: `session-handoff` / `세션 핸드오프`

`session-handoff 실행` 또는 `@session-handoff.md 실행`으로 호출한다.
세션 종료 전 다음 세션이 즉시 이어받을 수 있는 핸드오프 문서를 생성한다.

## 1. 저장 위치

```
_core/sessions/YYMMDD-HHMMSS_session-handoff.md
```

파일 저장 전 반드시 `date +%y%m%d-%H%M%S` 를 실행하여 실제 현재 시각을 확인한다.
플레이스홀더(`HHMMSS`, `000000` 등)를 그대로 쓰는 것을 금지한다.

## 2. 작성 원칙

- 다른 세션에서 이 문서만 읽어도 즉시 작업을 이어받을 수 있도록 작성한다.
- 확정된 내용과 미결 사항을 명확히 구분한다.
- 다음 작업은 AI에게 바로 지시할 수 있는 수준으로 구체화한다.
- 파일명·폴더명·결정사항은 정확하게 그대로 기록한다.
- 사용자의 표면적 요청과 배경 목적을 함께 기록한다.

## 3. 템플릿

```markdown
# [주제] 세션 핸드오프

> 작성일시: YYMMDD-HHMMSS
> 세션 목적: (한 줄)
> 이전 핸드오프: {파일명, 없으면 생략}

## 1. 세션 핵심 요약

(이 세션에서 논의된 항목을 불릿으로 — 확정/미결 구분 없이 모두 포함)

## 2. 사용자 요청 및 의도

이 세션에서 사용자가 요청한 내용과 그 배경 목적을 기록한다.

| 요청 내용 | 배경 목적 |
|---|---|
| ... | ... |

## 3. 확정된 결정사항

이 세션에서 최종 결정된 사항을 기록한다.

| 항목 | 확정 내용 | 비고 |
|---|---|---|
| ... | ... | ... |

## 4. 미결 사항

다음 세션에서 결정이 필요한 항목을 기록한다.

| # | 항목 | 현재 상태 | 결정 필요 내용 |
|---|---|---|---|
| 1 | ... | 검토 중 | ... |

## 5. 다음 작업 목록

다음 세션에서 수행할 작업을 우선순위 순으로 기록한다.

| 우선순위 | 작업 | 관련 파일 | 비고 |
|---|---|---|---|
| 1 | ... | ... | ... |

## 6. 다음 세션 시작 지시문

아래는 이전 세션의 컨텍스트입니다.
이 내용을 기반으로 [다음 작업]을 진행해 주세요.

참고 파일:
- 핸드오프: _core/sessions/YYMMDD-HHMMSS_session-handoff.md

## 7. 참고 자료

이 세션과 관련된 파일 및 외부 자료를 기록한다.

| 항목 | 경로 | 용도 |
|---|---|---|
| ... | ... | ... |
```
