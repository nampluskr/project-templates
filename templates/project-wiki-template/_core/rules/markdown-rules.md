# markdown-rules.md

이 프로젝트에서 작성하는 모든 마크다운 문서에 적용되는 필수 규칙이다.

## 1. 문체

- 본문은 서술체(`~이다`, `~한다`) 사용 — 경어체(`~입니다`, `~합니다`)는 문서 본문 금지
- 이모지 사용 금지
- 수학·기술 용어는 영어 단독 표기 (`batch normalization`, `learning rate`)
- 테이블·목록 바로 앞에 도입 문장 1문장 이상 삽입

## 2. 구조

- `---` 구분선 사용 금지
- 헤더 레벨 건너뜀 금지 (H2 다음 H4 직접 사용 불가)
- H4 이하(`#####` 등) 사용 금지

헤더 번호 체계는 다음과 같다.

| 레벨 | 용도 | 번호 형식 |
|---|---|---|
| H1 `#` | 문서 제목 | 없음 |
| H2 `##` | 섹션 | `1.`, `2.` |
| H3 `###` | 서브섹션 | `1.1.`, `1.2.` |

## 3. 파일명

- 단어 구분: 하이픈(`-`), 접두사·접미사 연결: 언더스코어(`_`)
- 날짜 형식: `YYMMDD-HHMMSS` (메타정보), `YYMMDD` (파일명)

## 4. Obsidian 최적화

### 4.1. 프론트매터

```yaml
---
tags: [tag1, tag2]
created: YYYY-MM-DD
updated: YYYY-MM-DD
---
```

### 4.2. 내부 링크

```markdown
[[문서명]]
[[문서명|표시텍스트]]
```

링크 방향은 상위→하위 단방향으로 제한한다.
